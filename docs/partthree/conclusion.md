# نتیجه‌گیری

> منبع: [Shape Up - Conclusion](https://basecamp.com/shapeup/3.7-conclusion)

شیپ‌آپ مجموعه‌ای از تکنیک‌های جداگانه نیست؛ یک نگاه منسجم به توسعه محصول است. ایده اصلی این است که پیش از شروع ساختن، کار را به اندازه کافی شیپ کنیم، سپس با شرط‌بندی محدود و زمان ثابت، مسئولیت واقعی را به تیم بدهیم.

## مفاهیم کلیدی

- ایده خام پروژه نیست.
- اشتهای زمانی جای تخمین را نمی‌گیرد؛ محدودیت خلاقانه‌ای برای طراحی راه‌حل است.
- کار شیپ‌شده باید زمخت، حل‌شده و مرزبندی‌شده باشد.
- پیچ، پروژه را برای شرط‌بندی آماده می‌کند.
- شرط یعنی تعهد محدود یک تیم برای یک چرخه، بدون مزاحمت.
- بک‌لاگ رسمی لازم نیست؛ ایده‌های مهم دوباره برمی‌گردند.
- تیم‌ها پروژه می‌گیرند، نه فهرست تسک.
- اسکوپ‌ها زبان و نقشه پروژه را می‌سازند.
- نمودار تپه‌ای عدم‌قطعیت و پیشرفت واقعی را نشان می‌دهد.
- زمان ثابت، اسکوپ متغیر و کیفیت حفظ‌شده، راه عرضه پایدار است.

## ارتباط

روش شیپ‌آپ با خواندن تمام نمی‌شود؛ باید آن را در مقیاس کوچک امتحان کرد، نتیجه را دید و با اندازه تیم و محصول خودتان تطبیق داد. بهترین نقطه شروع، انتخاب یک پروژه کوچک، شیپ کردن آن با اشتهای زمانی روشن و اجرای یک چرخه واقعی است.
